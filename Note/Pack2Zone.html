出力テープのレイアウト

Java環境では、ZONEAREA/KANAAREAは通常文字として、PACKAREAはバイト文字として取り込む必要があります。

というわけで、

  • ゾーン10進数の開始バイト/終了バイト
  • パック10進数の開始バイト/終了バイト

を別ファイルに持たせておきます（いわゆるCOPY句もどき）。

文字コード確認

> コンパイル後、モジュール実行
$ ./WRITE001
> テープ内容確認
$ cat OUT-TP.dat
あいう?⏎
> 文字コード確認
$ file --mime OUT-TP.dat
OUT-TP.dat: text/plain; charset=utf-8

ということで、出力テープはUTF-8（macOS標準文字コード）として出力されます。
ちなみに「えお」がちゃんと出ないのは、N(1)は2バイト確保するのだが、Unicodeの全角文字は3バイト使用します。つまり、領域が5バイト足りず、文字化けします（layout.csvでKANAAREAを1文字3バイト確保させたのはこのためです）。

ただ、今回はこれで文字コードがわかったため良しとします。

※ Windows日本語版環境の場合、Windows日本語版の標準文字コードはShift_JIS。
　Shift_JISの全角は2バイトなので、Nタイプでも文字数分領域が確保され、文字化けは発生しません（とネットにありました）。

Ph.2 パック10進数をJavaでゾーン10進数に変換する

TL;DR

コードが少し長いので、端的に処理内容を記載します。

  • COBOLで出力したテープは2バイトずつ読む
  • layout.csvは予めArrayList型に変換しておく
  • layout.csvより読んだテープの領域が
    • "Z"/"X"/"N"タイプ：char型に変換してそのまま出力
    • "P"タイプ：HEXをリテラル文字列にして出力

Javaソース

import java.io.*;
import java.util.ArrayList;

/**
 * Pack2Zone
 * This program is convert PackDecimal to ZoneDecimal.
 */
public class Pack2Zone {
    public static void main(String[] args) {
        // File path
        // (Class file is create into java dir).
        final String fname = "../OUT-TP.dat";
        // Layout file path
        final String layfname = "../layout.csv";
        try{
            // Open COBOL written tape.
            FileInputStream fis = new FileInputStream(fname);
            // Converted pack to zone (String).
            String Pack2Zone = "";
            // Create Layout array / Array index.
            ArrayList<ArrayList<String>> layouts = CSVtoArray(new File(layfname));
            int i = 0;
            // Read first byte / first layout.
            int b = fis.read();
            ArrayList<String> layout = layouts.get(i);
            // Read byte counter.
            int j = 0;
            // End Condition: File is end.
            while (b != -1) {
                j++;
                // Layout is out of range -> Read next layout.
                if (j > Integer.parseInt(layout.get(2))) {
                    layout = layouts.get(++i);
                }
                switch (layout.get(0)) {
                    case "Z":
                    case "X":
                    case "N":
                        // Zone/X is simple concat.
                        Pack2Zone += (char)b;
                        break;
                    case "P":
                        // Pack is concat hex str.
                        Pack2Zone = concatHexString(Pack2Zone, Integer.toHexString(b));
                        break;
                }
                // Read next byte.
                b = fis.read();
            }
            // Tape close.
            fis.close();
            // Convert ConcatHexString to int.
            System.out.println(Pack2Zone);
        } catch (Exception e) {
            // If exception, print exception infomation.
            e.printStackTrace();
        }
    }

    /**
     * Return concat hex string.
     */
    private static String concatHexString(String str, String hex) {
        // "f" or "c" only is 0.
        if (hex.equals("f") || hex.equals("c")) return str + "0";
        // "d" only is minus.
        if (hex.equals("d")) return "-" + str;
        // "f9" is unsigned, "c9" is plus for signed. ("9" is numeric)
        if (hex.contains("f") || hex.contains("c")) return str + hex.substring(1);
        // "d9" is minus for signed. ("9" is numeric)
        if (hex.contains("d")) return "-" + str + hex.substring(1);
        // other is simple concat.
        return str + hex;
    }

    private static ArrayList<ArrayList<String>> CSVtoArray(File file) {
        // Layout array.
        ArrayList<ArrayList<String>> layouts = new ArrayList<>();
        try {
            // Open layout csv.
            BufferedReader br = new BufferedReader(new FileReader(file));
            // Read first line.
            String line = br.readLine();
            while (line != null) {
                ArrayList<String> layout = new ArrayList<>();
                for (int i = 0; i < 3; i++) {
                    layout.add(line.split(",")[i]);
                }
                layouts.add(layout);
                line = br.readLine();
            }
            br.close();
        } catch (Exception e) {
            e.printStackTrace();
        }
        return layouts;
    }
}

まとめ

パック10進数はJavaでも無理やり処理しようとすれば処理できます。
ただ、Javaでは通常パック10進数は扱わないので、見ての通り、超非現実的なやり方です。
ですので、マイグレーションの際は、パック10進数はゾーン10進数に変換した上で取り込むのが理想です。

また、DBからパック10進数のデータを引っ張ってくると、もっと厄介な可能性があります。
